DKARL TUNNEL is a VPN and tunnel configuration app for advanced users. This page explains what information the app may process, how ads and consent work, and how users can manage their privacy choices.

Section 01

Information We Process

DATA_PROCESSING

The app may process configuration values entered by the user, including:

  • Server addresses, ports, and payloads
  • SNI / SSL values and DNS settings
  • Proxy settings and tunnel mode choices
  • Imported .dkl configuration files

These values are used only to run the VPN/tunnel features selected by the user. DKARL TUNNEL may also read basic device and network state needed for VPN operation — such as whether a network is connected, whether mobile data is active, and carrier/operator information when a config is locked to a specific mobile network or operator.

Section 02

VPN And Network Traffic

NETWORK

When the VPN is started, traffic may be routed through the server or tunnel configuration chosen by the user. DKARL TUNNEL does not provide the remote server itself — the privacy and logging behaviour of any remote server depends entirely on the server owner and the configuration used.

Section 03

Advertising And Consent

ADS / ADMOB

DKARL TUNNEL uses Google AdMob to display ads. Google and its partners may collect identifiers, device information, and ad interaction data to provide, measure, and improve advertising — subject to the consent choices shown in the app.

For users in regions where consent is required, DKARL TUNNEL uses Google's User Messaging Platform consent form before requesting ads. Ads are not requested until the Google consent status allows ad requests.

Section 04

Manage Or Revoke Consent

USER_CONTROL
Action required

Users can manage or revoke advertising consent inside the app by opening DKARL TUNNEL, going to the menu, and choosing Privacy options when shown. If the option is not visible, Google has not required a privacy options entry point for that user or region at that time.

This page serves as the privacy and revocation information link for AdMob: https://dkarl.site/privacy/#revoke-consent

Section 05

Configuration Files

.DKL FILES

Imported or exported .dkl configuration files may contain server credentials and tunnel settings. Users should only import files from trusted sources. Protected configs may restrict editing or exporting inside the app, but no offline app file can be guaranteed impossible to reverse engineer.

Section 06

Third-Party Services

DEPENDENCIES

The app may use the following services, each operating under its own privacy policy:

  • Google AdMob — advertising delivery and measurement
  • Google Play Services — core Android platform services
  • Google Play Review API — in-app review prompts
  • Other Android libraries for features, stability, and user experience
Section 07

Security

SECURITY

DKARL TUNNEL uses reasonable safeguards for local configuration handling. Users should avoid sharing sensitive configs and should only use trusted servers. VPN and tunnel security depends on the configuration and remote infrastructure selected by the user.

Section 08

Children

AGE_POLICY

DKARL TUNNEL is intended for advanced users and is not directed to children. Users who are not old enough to consent to data processing in their region should not use the app without a parent or guardian.

Section 09

Changes

UPDATES

This policy may be updated when the app, advertising setup, or legal requirements change. The effective date at the top of this page will be updated whenever material changes are made.

Section 10

Contact

SUPPORT

For privacy questions about DKARL TUNNEL, contact the developer through the support contact listed on the app's Google Play Store page.
